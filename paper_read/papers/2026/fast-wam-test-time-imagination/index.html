2. 研究背景

2.1 World Action Models (WAMs) 的兴起

World Action Models (WAMs) 作为 Vision-Language-Action (VLA) 模型的有力替代方案正在兴起。与 VLA 不同，WAMs 显式建模视觉观测在动作作用下的演化过程，这种显式的世界建模被认为对具身控制任务更有优势。

2.2 现有范式的痛点

Imagine-then-Execute 的问题

  • • 大多数现有 WAMs 采用"先想象后执行"范式
  • • 测试时需要迭代视频去噪（iterative video denoising）
  • • 导致 substantial test-time latency（通常 800ms+）
  • • 但：显式未来想象是否真的必要？这个问题从未被系统研究

2.3 核心研究问题

本文提出的核心问题是：

"WAMs 是否需要在测试时进行显式的未来想象？
还是其收益主要来自训练时的视频建模？"

2.4 WAM 的三大范式

Figure 1: 三种代表性 WAM 范式。(A) Joint-modeling WAMs 同时去噪未来视频和动作；(B) Autoregressive WAMs 自回归生成未来；(C) Fast-WAM 仅在训练时使用视频，测试时直接预测动作。

Part II · 方法详解

3. 方法详解：Fast-WAM

3.1 核心洞察

关键假设：视频预测在 WAMs 中的主要价值可能在于训练时改善世界表示，而非测试时生成未来观测。

3.2 模型架构

Figure 2(a): Fast-WAM 模型架构。采用 Transformer 架构，训练时同时预测未来视频帧和动作，但测试时仅使用动作预测分支。

架构关键设计

  • 基础架构：基于 Transformer 的扩散模型（Diffusion Model）
  • 输入：当前视觉观测 + 语言指令
  • 训练目标：联合去噪未来视频帧 + 预测动作序列
  • 测试行为：跳过视频去噪，直接生成动作
  • 关键技巧：使用 masking 策略分离训练和测试时的信息流动

Figure 2(b): 训练和推理时的 mask 设计。训练时使用联合 mask（预测视频+动作），推理时仅使用动作 mask。

3.3 受控变体设计

为了解耦"训练时视频建模"和"测试时未来生成"的作用，作者设计了多个受控变体：

变体 训练时视频 测试时视频 目的

Joint-modeling ✓ ✓ 传统 WAM（对比基线）
No-video ✗ ✗ 纯 VLA（验证视频训练的价值）

4. 实验评估

4.1 实验设置

LIBERO

经典的机器人操作基准，包含多种操纵任务

RoboTwin 2.0

双臂机器人操作任务，更具挑战性

Real-World

Galaxea R1 Lite 平台上的真实毛巾折叠任务

4.2 实现细节

基础模型：Diffusion Transformer

输入分辨率：256×256

动作维度：7-DoF 机械臂

训练数据：公开数据集（无 embodied pretraining）

推理设备：NVIDIA RTX 4090

扩散步数：10 steps（测试时）

4.3 主要结果

核心发现

  • • Fast-WAM 与 Joint-modeling WAMs 性能相当（在 LIBERO 和 RoboTwin 上）
  • • 而 No-video 变体性能显著下降（验证了视频训练的价值）
  • • Fast-WAM 4倍 faster（190ms vs 800ms+）
  • • 在真实世界任务上也表现良好

Part IV · 对比分析

5. 深度对比分析

🔍 为什么 Fast-WAM 能成功？

假设：视频训练改善世界表示

视频预测任务迫使模型学习环境的动态规律、物理约束和因果关系，这些知识编码在模型的潜在表示中，即使不生成视频也能用于动作预测。

假设：视频生成是"昂贵的副产品"

测试时的视频生成可能需要大量计算，但对动作质量的边际收益有限。模型已经"知道"该做什么，生成视频只是可视化这一过程。

假设：任务特性

对于短期动作预测任务，显式未来想象可能确实不必要。但对于长程规划，视频生成可能有更大价值（论文未验证此点）。

Part V · 批判性分析

6. 批判性分析与局限

1. 任务范围限制

实验主要集中在短期操作任务（single-step or few-step manipulation）。对于需要长程规划（multi-step reasoning）或探索（exploration）的任务，显式未来想象可能更有价值。论文未在复杂长程任务上验证。

2. "视频训练改善表示"缺乏深入验证

论文声称视频训练改善世界表示，但未提供表示分析（如 probing、visualization）来验证这一点。为什么视频训练能改善表示？改善了哪些方面？这些问题未得到回答。

3. 与 SOTA VLA 的公平对比

论文与 RT-2 等方法对比，但 RT-2 使用大规模互联网预训练，而 Fast-WAM 没有。这种对比是否公平？Fast-WAM 的优势是否主要来自架构设计，还是训练数据差异？

4. 消融实验深度不足

论文设计了受控变体，但未深入分析为什么移除视频训练会导致性能下降。是表示质量下降？还是动作预测头训练不充分？缺乏机制性分析。

5. 190ms 延迟的语境

虽然 190ms 比传统 WAMs 快 4 倍，但对于实时控制是否足够？人类反应时间约 200-250ms，190ms 接近临界值。论文未讨论这是否满足动态任务需求。

可能的未验证假设

  • • 假设：视频训练对所有类型的机器人任务都有帮助（未在导航、移动等任务上验证）
  • • 假设：跳过视频生成不会损失可解释性（传统 WAMs 可通过可视化预测来调试）
  • • 假设：模型已充分学习世界动态（对于 OOD 场景是否成立？）

7. 总结与评价

贡献与优点

  • • 提出了一个重要且未被充分研究的问题
  • • Fast-WAM 是一个简单有效的工程解决方案
  • • 受控实验设计清晰，结论有说服力
  • • 在仿真和真实世界都进行了验证
  • • 代码和项目页面已公开（可复现性好）

局限与不足

  • • 缺乏对"视频训练改善表示"的机制性分析
  • • 未在长程规划任务上验证
  • • 与 SOTA VLA 的对比可能不公平
  • • 消融实验深度不足
  • • 190ms 延迟的实用性未充分讨论

综合评价

Fast-WAM 是一篇问题导向的 solid work。它没有提出复杂的新架构，而是通过一个简单的问题（"我们真的需要测试时视频生成吗？"）挑战了现有范式。答案是：可能不需要。这一发现对 WAM 领域有重要启发——或许我们应该重新思考视频生成的定位：它是核心的推理工具，还是训练时的辅助任务？论文的工程贡献明确，但理论深度有待加强。总体而言，这是一篇值得阅读的论文，特别适合关注机器人学习效率和实时性的研究者。
