概览与背景

1 论文概览

🎯 核心贡献

  • ✓ 提出以动作中心的高效World-Action模型，训练时利用未来视觉动态提供密集监督
  • ✓ 推理时未来视频预测可选，支持纯动作解码模式，实现低延迟控制
  • ✓ 构建大规模机器人数据集进行预训练，将通用视频生成模型转化为机器人策略学习强初始化

💡 一句话总结

GigaWorld-Policy通过因果掩码设计实现了训练与推理的解耦：训练时同时学习动作预测和视频生成，利用像素级动态提供密集监督；推理时可跳过视频生成，仅解码动作，在保持性能的同时实现9倍推理加速。

2 研究背景

📊 领域现状

Vision-Language-Action (VLA)

基于VLM的VLA模型（如π₀、GigaBrain-0、X-VLA）通过大规模预训练实现强大的视觉-语言对齐和动作解码，但面临监督稀疏性问题——动作监督稀疏且多样性低，容易导致模型依赖浅层上下文信号。

World-Action Models (WAM)

如Motus、Cosmos-Policy等方法将视频生成模型融入机器人策略学习，通过预测未来视觉动态提供时间密集监督，但通常需要迭代采样生成视频，推理开销大且视频预测误差会传播到动作解码。

⚠️ 现有方法的核心瓶颈

1
推理开销大

联合推理未来视觉动态和对应动作需要大量计算，现有WAM方法需要迭代扩散采样来生成未来视频，导致高延迟。

2
表征纠缠

联合建模往往使视觉表征和运动表征纠缠，动作预测准确性过度依赖未来视频预测质量。视频预测误差会随时间累积，影响长期控制性能。

3 核心问题定义

问题形式化

机器人操作可形式化为序列决策任务。在每个时间步t，机器人接收多视角RGB观测 oₜ、自然语言指令 l 和本体感受状态 sₜ。策略需要预测长度为H的动作块 aₜ:ₜ₊ₕ。

传统VLA范式
p(a | o, s, l)

仅从演示中学习动作分布，无显式观测空间监督

GigaWorld-Policy范式
p(a | o, s, l) — 动作建模
p(o_future | o, s, a, l) — 视觉前向动态

同时建模动作分布和视觉动态，提供双重监督信号

研究假设与验证思路

  • H1: 未来视觉动态可作为密集监督信号和推理由信号，无需过度依赖显式视频合成
  • H2: 通过因果设计防止未来视频token影响动作token，可使视频生成在推理时变为可选
  • H3: 课程式预训练（Web视频 → 具身数据 → 任务数据）可提供更强的策略初始化

4 架构设计

整体架构

GigaWorld-Policy基于5B参数扩散Transformer（Wan 2.1），通过以动作为中心的目标预训练作为机器人操作的World-Action模型。核心设计包括多视角输入融合、共享Transformer块和因果自注意力掩码。

输入Token设计

多视角融合

为支持多视角生成且不修改backbone，将三个相机视角合并为单一复合图像：

oₜ^comp = Compose(oₜ^left, oₜ^front, oₜ^right)

该复合表示保留各视角在共享坐标系中的空间结构，促进跨视角一致性。

稀疏未来帧预测

由于相邻观测具有强时间连续性和冗余性，采用固定步长仅预测稀疏的未来帧集合：

{oₜ₊ₖΔ^comp}ₖ₌₁ᴷ

沿动作horizon每Δ步预测一帧未来观测，保留场景关键演变同时减少监督冗余。

共享Transformer块

与MoE设计不同，所有token类型由单一共享Transformer块栈处理：

  • • 所有token在每层共享相同的Q/K/V投影矩阵
  • • 紧密耦合动作token与视觉证据，同时保持预训练backbone的计算特性
  • • 不同类型token使用不同位置编码：视觉token使用2D图像网格位置编码，状态和动作token使用1D时间位置编码

6 数学机制

因果自注意力机制

为统一动作生成和前向视觉动态建模，将所有模态打包为单一token序列，使用因果注意力掩码控制信息流：

统一Token序列
T_o ; T_s ; T_a ; T_f
T_t = [T_o; T_s; T_a; T_f]
块级因果注意力掩码规则
✓ T_s ↔ T_o: 状态和当前观测可相互关注，但不能关注T_a或T_f
✓ T_a → {T_s, T_o}: 动作token可关注状态/观测，但不能关注T_f
✓ T_f → {T_s, T_o, T_a}: 未来视频token可关注所有前置token，实现基于动作块的前向动态预测

关键设计：语言指令l不纳入统一自注意力序列，而是作为外部条件信号通过交叉注意力提供，因此不参与上述因果排序。

Flow Matching训练目标

使用Flow Matching优化动作预测和视觉前向动态建模：

插值噪声变量 (Eq. 6)
x_t = (1-t)·x_0 + t·x_1 + σ_t·ε
目标速度场 (Eq. 7)
v_t = x_1 - x_0
视频流匹配目标 (Eq. 8)
L_video = E[||v_f - φ_θ(z_f | history, a)||²]
动作流匹配目标 (Eq. 9)
L_action = E[||v_a - φ_θ(z_a | history)||²]
组合损失 (Eq. 10)
L = λ_video · L_video + λ_action · L_action
  • • 预训练阶段：仅优化视频流匹配目标
  • • Post-training阶段：组合视频和动作目标，权重λ平衡贡献
  • • 强调动作预测同时保留视频一致性正则化

7 关键创新深入解析

创新点1: 可选视频生成的因果设计

L1: 直观理解

就像一个学生在解题时，草稿纸（视频生成）帮助思考过程，但交卷时只需要答案（动作）。GigaWorld-Policy训练时用"草稿纸"辅助，推理时可以直接"交卷"，不需要写草稿。

L2: 机制流程
训练阶段
o + s → a (动作预测)
o + s + a → f (视频预测)
两者并行训练，共享梯度
推理阶段
o + s → a ✓ (仅需此步)
f 生成可选
跳过视频token，9×加速
L3: 技术细节
  • • 因果掩码的关键作用： 通过限制T_a只能关注{T_s, T_o}，确保动作预测不依赖未来视频token
  • • 解耦原理： T_f可以关注T_a，但T_a不能关注T_f，形成单向信息流
  • • 推理优化： 不实例化任何T_f token，仅采样动作块，避免视频生成开销
  • • KV Cache复用： 如需视频生成，可复用动作去噪阶段的KV Cache，条件化地去噪视频token

⚠️ 常见误区澄清

误区 澄清
"跳过视频生成会导致信息丢失" 因果设计确保动作token从不依赖视频token，训练已完成解耦
"共享参数意味着共享表示" 虽然Q/K/V矩阵共享，但不同类型token使用不同位置编码，保持结构区分
"稀疏未来帧预测会损失信息" 实验表明密集预测无额外收益，适度稀疏足以提供有用的时序先验
