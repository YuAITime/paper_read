DeepSeek-OCR: Contexts Optical Compression

Haoran Wei, Yaofeng Sun, Yukun Li

[Submitted on 21 Oct 2025]

Computer Vision and Pattern Recognition (cs.CV)
Part I · 概览与背景

核心贡献

  • 1首次系统量化分析视觉-文本token压缩比：10×压缩达97%精度，20×压缩仍保持60%
  • 2提出DeepEncoder架构：380M参数，通过窗口注意力+全局注意力+16×卷积压缩器实现低激活高压缩
  • 3工业级实用价值：单卡A100日处理20万+页，OmniDocBench上100 tokens超越GOT-OCR2.0(256 tokens)
  • 4开源贡献：代码和模型权重公开发布
摘要: We present DeepSeek-OCR as an initial investigation into the feasibility of compressing long contexts via optical 2D mapping...

2. 研究背景与动机

核心洞察

LLM处理长文本时面临序列长度的二次方复杂度增长。但一个包含文档文本的图像可以用比等效数字文本少得多的token表示丰富的信息——这提示视觉模态可作为文本信息的高效压缩介质。

深度分析：为什么是"光学压缩"？

传统思路：文本→文本（直接压缩，如BPE、SentencePiece）
本文思路：文本→图像→视觉token（光学2D映射压缩）

关键优势：

  • 图像天然保留2D空间结构（段落、表格、图表位置关系）
  • 视觉编码器（如ViT）已将图像压缩为少量token（如196个patch）
  • 人眼阅读正是"光学解码"过程，证明信息密度可行

现有VLM的局限

当前开源VLM的视觉编码器存在三大问题：(1)双塔架构（如Vary）部署复杂，难以并行；(2)单塔架构（如Qwen2-VL）激活内存高；(3)所有方法视觉token数量过多，无法灵活控制压缩比。

3. 核心问题定义

研究假设

给定N个文本token，能否用远少于N个视觉token（如N/10或N/20）通过光学2D映射编码，再由解码器高精度恢复原文？

形式化定义：

压缩比 = 文本token数 / 视觉token数

目标：在压缩比≥10时，OCR解码精度≥95%

关键挑战

  • 1. 高分辨率处理：文档图像通常1024×1024或更高，需要编码器支持高分辨率且保持低激活
  • 2. 可变token数：不同文档密度不同，需要支持多种分辨率模式（64/100/256/400/600/800 tokens）
  • 3. 信息保真：压缩过程中保留文字内容、布局结构、图表关系
  • 4. 端到端训练：编码器和解码器需要联合优化

5. DeepEncoder 深度解析

设计哲学：低激活 + 高压缩 + 多分辨率

DeepEncoder需要满足5个要求：(1)处理高分辨率；(2)高分辨率下低激活；(3)少量视觉token；(4)支持多分辨率输入；(5)参数量适中。

双组件架构

组件1：窗口注意力（Window Attention）

基于SAM-base (patch-size 16)，负责视觉感知特征提取。使用局部窗口注意力降低激活内存。

组件2：全局注意力（Global Attention）

基于CLIP-large (patch-size 14)，负责视觉知识特征提取。使用密集全局注意力捕获长距离依赖。

16× 卷积压缩器

两个组件通过16×卷积压缩器串联，将高分辨率特征图压缩为少量token。

输入图像 (1024×1024) → SAM-base (patch 16) → 64×64特征图 → 16×卷积压缩器 → 4×4 = 16个token (最低分辨率模式)

动态位置编码插值

为支持多分辨率（64/100/256/400/600/800 tokens），DeepEncoder使用动态位置编码插值。不同分辨率模式同时训练，使单个模型支持多种token数输出。

6. 训练方法论

两阶段训练策略

阶段1：独立训练DeepEncoder

使用紧凑语言模型，以next token prediction框架训练。数据：OCR 1.0 + OCR 2.0 + 100M LAION通用数据。batch size 4096，2 epochs。

阶段2：端到端训练DeepSeek-OCR

使用Section 3.4的数据训练完整模型。在HAI-LLM平台上使用pipeline并行（PP），分4部分：Encoder 2部分 + Decoder 2部分。

Gundam-master模式

在预训练DeepSeek-OCR基础上，使用6M采样数据继续训练得到的增强模式。

关键洞察

DeepSeek-OCR的核心优势在于可控压缩比：根据文档类型和精度需求动态调整视觉token数。Slides用64 tokens，复杂论文用800 tokens，实现资源的最优分配。

11. 消融实验

分辨率模式消融

DeepEncoder支持6种分辨率模式：64/100/256/400/600/800 tokens。实验表明：

  • • 64 tokens：Slides足够，复杂文档不足
  • • 100 tokens：书籍/报告的最佳平衡点
  • • 256+ tokens：论文/表格等密集文档需要

组件消融

SAM-base（窗口注意力）+ CLIP-large（全局注意力）的串联设计：

  • • 仅SAM：局部特征好，全局语义弱
  • • 仅CLIP：全局语义好，高分辨率激活爆炸
  • • SAM+CLIP+16×压缩：兼顾两者，实现低激活高压缩
Part V · 深入讨论与批判

12. 深入讨论

Deep Parsing：二次调用能力

DeepSeek-OCR不仅做OCR，还能通过"二次模型调用"深度解析文档中的图像：

图表解析

自动识别柱状图、折线图、饼图，提取数据表格

化学公式

识别分子结构，转换为SMILES表示

几何图形

解析平面几何图，提取点线面关系

自然图像

文档中的照片生成密集caption

多语言能力

训练数据覆盖约100种语言，中文和英文占主导（25M/30M）。对于PDF中的多语言文档，DeepSeek-OCR支持布局解析和OCR的联合输出。

与人类记忆的类比

论文提出一个有趣的类比：人类记忆随时间衰退，视觉感知随空间距离退化——两者表现出相似的渐进模式。这启发了一个未来方向：

渐进式压缩：对于 older contexts，可以逐步缩小渲染图像尺寸以进一步减少token消耗。近期上下文保持高分辨率，远期上下文降低分辨率。

LLM/VLM训练数据构建

DeepSeek-OCR的工业级应用：为LLM/VLM预训练构建数据。单卡A100-40G日处理20万+页，可大规模生成高质量训练语料。

13. 批判性分析

核心创新点评析

光学2D映射确实是创新点，但技术实现上并非全新：SAM+CLIP的组合在VLM中已有先例（如LLaVA），16×卷积压缩也是标准操作。真正的贡献在于系统性的压缩比-精度权衡分析和工业级落地。

局限性质疑

1. 压缩比边界的真实含义

20×压缩时60%精度看似可用，但实际意味着什么？是每10个字错4个，还是关键信息（如数字、名称）更容易出错？论文未详细分析错误类型分布。

2. 泛化能力的疑问

OmniDocBench覆盖的文档类型有限。对于手写体、低质量扫描件、复杂数学公式、多栏混排等场景，性能如何？100种语言中，非拉丁文字（中文、阿拉伯文、印地文）的表现是否一致？

3. 与通用VLM的兼容性

DeepSeek-OCR的压缩视觉token是否能被通用VLM（如GPT-4V、Claude）理解？还是只能配合专用解码器？如果无法通用，应用场景将受限。

4. 训练数据的版权问题

30M页PDF来自互联网，包含大量 copyrighted 内容。用于训练是否合规？商业使用时的法律风险如何？

5. 与文本压缩方法的对比缺失

论文未与纯文本压缩方法（如BPE、SentencePiece的变体，或专门的文本压缩算法如LZ4、Brotli）对比。视觉压缩是否比文本压缩更高效？

常见误区澄清

误区澄清
"压缩=信息丢失"是有损压缩，但保留了关键语义和布局结构。10×压缩97%精度证明信息损失可控。
"只能做OCR"实际支持deep parsing（图表、公式、几何）、多语言处理、通用图像理解。
"替代传统OCR"当前是互补关系。传统OCR（如MinerU）在精度要求极高时仍占优，DeepSeek-OCR在长文档/大数据量场景更优。
"视觉token通用"当前视觉token需要专用解码器，与通用VLM的兼容性未验证。

实用价值

核心贡献：首次系统验证视觉-文本压缩的可行性边界，10×压缩97%精度为LLM长上下文处理开辟新路径。

适用场景：

  • 大规模文档数字化（图书馆、档案馆、企业文档库）
  • LLM/VLM训练数据构建（20万+页/天产能）
  • RAG系统文档预处理（降低token成本，加速检索）
  • 移动端/边缘设备文档理解（低带宽、低算力场景）
  • 多语言文档统一处理（100种语言支持）

风险提示：

  • 20×压缩时精度降至60%，高压缩比场景需谨慎评估
  • 手写体、低质量扫描件、复杂公式等场景未充分验证
  • 与通用VLM的兼容性限制应用范围
  • 训练数据的版权合规性需要关注

总体评价：这是一篇工程导向的优秀工作。虽然理论创新有限，但在系统性实验、工业落地和开源贡献方面表现突出。对于需要处理大规模文档的工业界应用，具有重要的参考价值。未来若能解决与通用VLM的兼容性、提升高压缩比下的精度，将具有更大的影响力。
