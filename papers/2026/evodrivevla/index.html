研究背景: VLA 模型在自动驾驶中的挑战

1

视觉表征退化问题 (Degradation-Adaptation Dilemma)

Vision-Language-Action (VLA) 模型在自动驾驶任务中面临一个核心困境：解冻视觉编码器 (Visual Encoder) 进行微调后，虽然可以增强对自动驾驶场景的感知能力，但同时会导致预训练阶段学到的通用视觉表征退化，降低模型的泛化能力。这被称为"退化-适应困境"。

2

长期规划不稳定性

VLA 模型在进行长时域轨迹规划时，容易出现误差累积问题，导致预测轨迹在时间和空间上不稳定。现有知识蒸馏方法多采用单轨迹蒸馏或多轨迹蒸馏，但未能有效解决规划能力的实质性提升问题。

3

现有蒸馏方法的局限

传统的单轨迹蒸馏 (如 DiMA) 直接使用教师模型预测的轨迹监督学生；多轨迹蒸馏 (如 DistillDrive) 通过构建规划词汇表增加多样性，但受限于预定义词汇，无法真正适应动态场景。

核心问题定义

Degradation-Adaptation Dilemma

视觉编码器在监督微调 (SFT) 阶段是否应该完全解冻？

  • 解冻 → 增强领域适应能力，但可能退化预训练表征
  • 冻结 → 保持通用表征，但限制任务特定感知能力
  • 目标: 增强自动驾驶场景感知，同时保持原始感知能力

Long-term Planning Instability

长期轨迹预测中的误差累积和不稳定性问题

  • 教师模型与学生模型在相同设置下训练时，规划能力无实质差异
  • 单轨迹蒸馏无法提供丰富多样的规划知识
  • 预定义规划词汇表限制了适应真实动态场景的能力

EvoDriveVLA 框架包含两个核心组件：Self-Anchored Visual Distillation (左) 通过 Self-Anchor Teacher 提供视觉锚定约束，利用 AnchorFormer 进行轨迹引导的关键区域感知；Oracle-Guided Trajectory Distillation (右) 利用未来感知信息构建 Oracle Teacher，通过粗到精轨迹优化和 MC-Dropout 采样生成高质量轨迹候选。

Self-Anchor Teacher

  • • 学生模型视觉编码器微调前的副本
  • • 训练期间权重保持冻结
  • • 提供稳定的预训练视觉表征作为锚点
  • • 配合 AnchorFormer 实现轨迹引导的 token 级蒸馏

Oracle Teacher

  • • 与学生模型相同的 Qwen2.5-VL 3B 架构
  • • 输入未来 T 秒的图像和自车状态（特权信息）
  • • 具备粗到精 (Coarse-to-Fine) 轨迹优化能力
  • • 通过 MC-Dropout 采样生成多样化轨迹候选

AnchorFormer 结构详解

Input Tokens
Visual Tokens $z_v$
Instruction $z_p$ + State $z_s$
Ground-truth Trajectory $z_{w^*}$
Learnable Query $q$

AnchorLayer

与 LLM 单层解码器共享架构，处理拼接后的观测 tokens、轨迹 tokens 和可学习查询 tokens，输出更新后的表征。

AnchorScorer

单层线性层，计算视觉 tokens 与查询 tokens 的 Hadamard 积，输出每个视觉 token 的锚定分数。

$S_a = \text{AnchorScorer}(\tilde{z}_v^t \odot \tilde{q})$

$W_a = \frac{1}{1 + \exp(S_a / \tau_v)}, \quad \tau_v = 2.0$

数学机制详解

L3 技术细节

视觉蒸馏损失 (Self-Anchored Visual Distillation Loss)

$$L_a = \frac{1}{N_v} \sum_{i=1}^{N_v} W_a^{(i)} \cdot \|z_v^{tea(i)} - z_v^{stu(i)}\|_2^2$$

符号说明:
  • • $N_v$: 视觉 token 数量
  • • $z_v^{tea}$: Self-Anchor Teacher 视觉表征
  • • $z_v^{stu}$: Student 视觉表征
  • • $W_a^{(i)}$: 第 $i$ 个 token 的锚定权重
关键特性:
  • • Token 级加权 MSE 损失
  • • 轨迹引导的关键区域感知
  • • 高权重区域施加更强锚定约束
  • • 权重 $\lambda_a = 0.05$
L3 技术细节

轨迹蒸馏损失 (Oracle-Guided Trajectory Distillation Loss)

隐层对齐损失 $L_h$

$$L_h = \frac{1}{N_t} \sum_{i=1}^{N_t} \|h_{stu}^{(i)} - h_{\hat{k}}^{(i)}\|_2^2$$

学生隐层与最优轨迹隐层的 MSE 对齐

Logits KL 散度 $L_l$

$$L_l = KL(\text{softmax}(l_{\hat{k}}/\tau_t) \| \text{softmax}(l_{stu}/\tau_t))$$

温度缩放后的预测分布 KL 散度, $\tau_t = 5$

最优轨迹选择:

$$\hat{k} = \arg\min_{l_k \in S_l} L_{CE}(l_k, W^*)$$

从候选集 $S_l$ 中选择与真值轨迹交叉熵损失最小的最优轨迹

L3 技术细节

总体训练损失

$$L_{all} = L + \lambda_a \cdot L_a + \lambda_h \cdot L_h + \lambda_l \cdot L_l$$

0.05 $\lambda_a$
0.1 $\lambda_h$
0.2 $\lambda_l$

L1/L2/L3 三层解释

L1 直观理解 类比解释

Self-Anchored Visual Distillation

"就像学生学习绘画时，老师不仅教他画汽车，还提醒他不要忘记基础的素描技巧"

Self-Anchor Teacher 相当于学生"过去的自己"，保留原始的视觉理解能力。AnchorFormer 就像一个"注意力放大镜"，根据轨迹信息判断哪些视觉区域更重要，对这些区域施加更强的"记忆保护"。

Oracle-Guided Trajectory Distillation

"就像有预知能力的教练，能看到未来几秒的路况，指导学生走最佳路线"

Oracle Teacher 拥有"未来视角"（特权信息），能生成更准确的轨迹。粗到精优化就像"草图→精稿"，MC-Dropout 采样则像"多角度尝试"，从中选出最好的方案教给学生。

L2 机制流程 数据流图

Self-Anchored Visual Distillation 数据流

1
复制 Student Visual Encoder → Self-Anchor Teacher (冻结)
2
输入: 多视角图像 $I^t$ → Teacher & Student 编码器
3
AnchorFormer: 指令 + 自车状态 + 真值轨迹 → 生成锚定权重 $W_a$
4
加权 MSE: $W_a \cdot \|z_v^{tea} - z_v^{stu}\|^2$ → 约束视觉表征

Oracle-Guided Trajectory Distillation 数据流

1
输入: 当前观测 + 未来 T 秒信息 → Oracle Teacher
2
Coarse-to-Fine: 粗轨迹 $W_t^c$ → 精轨迹 $W_t^f$
3
MC-Dropout: N=10 次采样 → 扩展候选集 $S_h, S_l$
4
选择最优轨迹 → 隐层对齐 $L_h$ + Logits KL $L_l$

关键设计决策
  • Self-Anchor Teacher 权重完全冻结，仅优化 AnchorFormer
  • Oracle Teacher 与学生共享架构，但输入未来信息作为特权
  • MC-Dropout 仅应用于隐层，通过轻量级 lm_head 计算 logits，开销极小
  • 权重设置: $\lambda_l > \lambda_h > \lambda_a$，强调轨迹分布学习

V

批判性分析

深入剖析：为什么有效？

1. 双教师协作机制

Self-Anchor Teacher 解决视觉 encoder 退化问题，Oracle Teacher 提供高质量轨迹监督。 两者协同工作，分别针对感知和规划两个层面的瓶颈。

2. Token 级细粒度蒸馏

AnchorFormer 实现轨迹引导的关键区域感知，对不同空间区域分配自适应锚定权重， 相比样本级蒸馏更精准。

3. 迭代精炼 + 多样性采样

Coarse-to-Fine 机制模拟轨迹渐进演化，MC-Dropout 在不增加模型参数的情况下 显著提升候选轨迹质量和多样性。

4. 隐藏状态 + Logits 双对齐

不仅复制 Oracle Teacher 的输出，更内化其复杂的轨迹修正推理能力， 实现更深层的知识迁移。

局限性质疑

1

Oracle Teacher 的公平性问题

质疑点： Oracle Teacher 使用了未来场景图像和自车状态作为特权信息，这实际上是一种"作弊"。 虽然学生模型在推理时不使用这些信息，但教师模型的优越性能是否真的能转化为学生的真实能力？

" 论文中 Oracle Teacher 的 L2 误差 (0.14/0.20) 远低于任何现有方法， 这种"超人类"性能是否会导致学生模型对不真实的目标过拟合？

2

MC-Dropout 的计算开销

质疑点： 论文声称 MC-Dropout "仅对隐藏状态应用，通过轻量级 lm_head 计算 logits，开销最小"。 但实际训练中，每次迭代都需要进行多次前向传播（采样 K 个隐藏状态），这对训练效率的影响如何？

论文声明

"minimal overhead"

实际考虑

K 次采样 × 每个 iteration 的开销

3

超参数敏感度

质疑点： 论文使用 λ_a=0.05, λ_h=0.1, λ_l=0.2 的固定权重组合，但没有进行超参数敏感性分析。 这些权重在不同数据集、不同模型规模下是否仍然最优？

λ_a (Visual KD) 0.05 无消融数据
λ_h (Hidden States) 0.1 无消融数据
λ_l (Logits) 0.2 无消融数据

常见误区澄清

≠ Self-Anchor vs Oracle Teacher

❌ 误区

两者都是"教师模型"，作用相同

✅ 正解

  • • Self-Anchor: 固定预训练权重，防止视觉退化
  • • Oracle: 使用未来信息，提供轨迹监督
  • • 一个是"锚点"，一个是"导师"

≠ 视觉蒸馏 vs 轨迹蒸馏

❌ 误区

两者都是传递知识，可以互相替代

✅ 正解

  • • 视觉蒸馏: 解决 encoder 微调退化问题
  • • 轨迹蒸馏: 提升规划精度和多样性
  • • 分别针对感知层和决策层

改进方向

更公平的教师模型

探索不依赖未来信息的强教师模型，或使用世界模型生成合理的未来预测作为替代。

效率优化

研究 MC-Dropout 的最优采样次数，或探索其他低成本的轨迹多样性增强方法。

自适应蒸馏权重

引入动态权重调整机制，根据训练阶段或样本难度自动调整 λ_a, λ_h, λ_l。
