Token拼接与共享机制深度解析

视频token和动作token究竟如何拼接？共享参数是什么意思？

核心疑问

1. 视频token和动作token是怎么拼接的？直接拼在一起吗？

2. "共享参数"到底是什么意思？是共享Transformer层吗？

3. 训练时注意力机制怎么处理这种混合序列？

4. 损失函数怎么同时计算视频和动作的loss？

1. 输入序列的具体构成

假设：

  • • 输入帧数：2帧（VAE编码后每帧有N个token）
  • • 预测未来：2帧视频 + 4个动作
  • • 每帧视频：假设16×16 = 256个patch token（实际可能更多）
  • • 每个动作：32维向量（论文中32个时间步 × 32维）

训练时的完整序列（简化示意）：

位置编码: 0   1   2   ...  255  256  257  ...  511  512  513  514  515
v₁ v₂ v₃ ... v₂₅₆ v₂₅₇ v₂₅₈ ... v₅₁₂ a₁ a₂ a₃ a₄
视频帧1 (256 tokens) 视频帧2 (256 tokens) 动作序列 (4 tokens)

关键：视频token和动作token确实是在同一个序列中拼接的！ 它们共享相同的位置编码空间，一起输入Transformer。

2. "共享参数"到底是什么意思？

架构图示意：

输入序列
视频token + 动作token
Token Embedding
共享同一个Embedding矩阵
Transformer Layers (5B参数)
• Self-Attention (Q, K, V投影矩阵)
• Feed-Forward Network
• LayerNorm
⬇ 所有token共享这些参数！
视频输出头
预测视频token
动作输出头
预测动作token

共享什么？

  • ✓ Token Embedding：同一个矩阵把token ID映射到向量
  • ✓ 所有Transformer层：Self-Attention和FFN参数完全共享
  • ✓ 位置编码：视频和动作使用同一套位置编码

不共享什么？

  • ✗ 输出头：视频和动作有各自的输出投影矩阵（Action Expert）

3. 训练时注意力机制如何处理混合序列？

这是最关键的部分！注意力机制让视频token和动作token互相影响：

Self-Attention 可视化

V₁ V₂ V₃ A₁ A₂
V₁ 💪 ⚡ ⚡ ✨ ✨
V₂ ⚡ 💪 ⚡ ✨ ✨
A₁ ✨ ✨ ✨ 💪 ⚡
A₂ ✨ ✨ ✨ ⚡ 💪
💪 强注意力 ⚡ 视频-视频交互 ✨ 视频-动作交叉

这意味着什么？

  • • 动作token可以看到所有视频token → 动作预测基于完整的视觉理解
  • • 视频token也可以看到动作token → 视频预测考虑了动作意图
  • • 两者在同一个表示空间中交互 → 学习统一的"世界+动作"表示

4. 损失函数如何计算？

联合损失函数：

Ltotal = λvideo · Lvideo + λaction · Laction

Lvideo 视频损失

  • • 计算预测的video token与真实video的MSE
  • • 只作用于被mask的未来视频位置
  • • 使用VAE的latent space计算

Laction 动作损失

  • • 计算预测的动作与真实动作的MSE
  • • 只作用于被mask的未来动作位置
  • • 通常在关节空间计算

梯度如何传播？

Lvideo的梯度 → 反向传播到视频输出头 → 传播到共享Transformer层

Laction的梯度 → 反向传播到动作输出头 → 传播到同一个共享Transformer层

→ 两个损失都更新共享参数，实现联合学习！

5. 推理时发生了什么变化？

训练时

V 输入：当前视频
? 预测：未来视频（必需！）
? 预测：未来动作

推理时 ⭐

V 输入：当前视频
— 跳过：不预测视频！
? 预测：未来动作

为什么可以跳过？

因为训练时视频预测的梯度已经更新了共享Transformer层， 这些层现在"知道"世界的物理规律。推理时，动作预测可以直接利用这些知识， 不需要再显式地预测视频来"复习"物理规律。

总结

1. Token拼接：视频和动作token在同一个序列中，共享位置编码

2. 共享参数：Embedding层和所有Transformer层完全共享

3. 注意力交互：视频和动作token通过Self-Attention互相影响

4. 联合损失：两个loss都反向传播到共享参数，实现联合训练

5. 推理加速：共享参数已内化知识，可以跳过视频预测
