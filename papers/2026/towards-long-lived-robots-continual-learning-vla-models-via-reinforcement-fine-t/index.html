Towards Long-Lived Robots: Continual Learning VLA Models via Reinforcement Fine-Tuning

Yuan Liu, Haoran Li, Shuai Tian, Yuxing Qin, Yuhui Chen et al.

[Submitted on 11 Feb 2026]

Robotics (cs.RO)
Part I · 概览与背景

核心贡献

  • 1提出 LifeLong-RFT：首个无需环境交互的 VLA 强化微调策略，结合 chunking-level on-policy RL 与 MDPR 机制
  • 2设计 MDPR (Multi-Dimensional Process Reward)：三维奖励机制 (QACR+CTAR+FCR) 量化动作块的多维贡献
  • 3LIBERO 持续学习基准上比 SFT 提升 22% 平均成功率，仅用 20% 训练数据即可有效适应新任务
摘要: Pretrained on large-scale and diverse datasets, VLA models demonstrate strong generalization... (VLA模型在大规模多样化数据集上预训练后展现出强泛化能力，但SFT需要大量任务特定数据且易灾难性遗忘。本文提出 LifeLong-RFT，通过 MDPR 三维奖励机制实现无需环境交互的强化微调，在持续学习任务上取得显著提升。)

2. 研究背景

领域定位

本研究属于 Robotics (cs.RO) 领域，聚焦 Vision-Language-Action (VLA) 模型的后训练 (Post-Training) 与持续学习 (Continual Learning)。

研究动机

  • 问题1 - 数据依赖：SFT 需要大量任务特定数据，限制了 VLA 在低数据/少样本场景下的快速适应能力
  • 问题2 - 灾难性遗忘：学习新技能时会退化先前获得的知识，无法支持 VLA 进化为长寿命智能体
  • 核心挑战：提升数据高效适应性往往会加剧遗忘，而保留先验知识又限制了从有限新数据中进行有效学习

解决思路

近期 LLM 研究发现 on-policy 强化学习对遗忘具有更强的鲁棒性。本文探索如何将其应用于 VLA 基础模型，核心挑战在于设计高效、可靠、可扩展的奖励信号。

3. 核心问题定义

研究假设

通过设计无需环境交互的多维过程奖励机制 (MDPR)，结合 chunking-level on-policy 强化学习，可以实现 VLA 模型的持续学习，同时解决数据依赖和灾难性遗忘问题。

验证思路

  1. 在 multi-task 学习场景下验证 LifeLong-RFT 相比 SFT 的性能提升
  2. 在 continual learning 场景下验证 forward transfer 和 backward transfer 指标
  3. 通过消融实验验证 MDPR 三个维度的各自贡献
  4. 在真实机器人环境中验证 sim-to-real 迁移能力

6. 关键创新深入解析：MDPR 三维奖励机制

🎯

L1 · 直观理解

MDPR 像一位"三维裁判"：QACR 检查动作 token 是否猜对，CTAR 检查实际运动轨迹是否准，FCR 检查输出格式是否正确。三者共同确保 VLA 既能准确预测又能正确执行。

🔄

L2 · 机制流程

1. VLA 采样动作块 → 2. QACR 比对 token 一致性 → 3. CTAR 解码后比对轨迹偏差 → 4. FCR 检查格式有效性 → 5. 加权合成 MDPR → 6. GRPO 计算优势并更新策略

🔬

L3 · 技术细节

QACR: Σ_i I(â_i = a_i) / |valid|
CTAR: (1/T)Σ_t[α·exp(-β||p̂_t-p_t||) + I(ĝ_t=g_t)]
FCR: I(valid format)
MDPR: ω·QACR + (1-ω)·CTAR + λ·FCR

QACR - 量化动作一致性奖励

逐位置匹配预测动作 token 与 ground truth，确保离散动作空间中的准确预测。关键：只有格式验证通过的序列才参与计算。

CTAR - 连续轨迹对齐奖励

将 token 解码为连续动作块后，计算与参考轨迹的空间偏差。pose 部分用指数衰减函数，gripper 用二元奖励，加权平均。

FCR - 格式合规奖励

二进制奖励验证输出是否符合 Fast+ tokenizer 的格式要求（动作维度、预测范围等），确保动作可执行。

14. 数学推导与算法详解

14.1 GRPO 目标函数推导

14.1.1 策略梯度基础

策略梯度定理指出，策略参数的梯度为：

∇_θ J(θ) = E[∇_θ log π_θ(a|s) · Q^π(s,a)]

其中 Q^π(s,a) 是状态-动作值函数，估计从(s,a)出发的期望累积奖励。

14.1.2 Group Relative Advantage 计算逻辑

GRPO 的核心创新：无需 Critic 网络，通过组内比较估计优势：

对于每个 query，采样 G 个输出 {o_1, o_2, ..., o_G}

计算组内均值：μ = (1/G) Σ_{i=1}^G r_i

计算组内标准差：σ = √[(1/G) Σ_{i=1}^G (r_i - μ)²]

相对优势：Â_i = (r_i - μ) / (σ + ε)

ε 是小常数防止除零。优势归一化使得不同 query 的梯度更新尺度一致。

14.1.3 Clip 机制的作用

防止策略更新过大，确保训练稳定性：

L^{CLIP}(θ) = E[min(r_t(θ)·Â_t, clip(r_t(θ), 1-ε, 1+ε)·Â_t)]

其中 r_t(θ) = π_θ(a_t|s_t) / π_{θ_old}(a_t|s_t)

当 Â_t > 0：只增加 r_t(θ) 到 1+ε，防止过度乐观
当 Â_t < 0：只减少 r_t(θ) 到 1-ε，防止过度惩罚

14.1.4 KL 散度正则化

防止策略偏离参考模型太远（灾难性遗忘）：

D_KL(π_θ || π_ref) = Σ_a π_θ(a|s) · log(π_θ(a|s) / π_ref(a|s))

完整的 GRPO 目标函数：

J_GRPO(θ) = (1/G) Σ_{i=1}^G [min(r_i·Â_i, clip(r_i)·Â_i) - β·D_KL(π_θ||π_ref)]

14.2 QACR 公式详解

输入/输出定义

  • • 输入: 预测动作 tokens â = [â_1, â_2, ..., â_T]
  • • 参考: Ground truth tokens a = [a_1, a_2, ..., a_T]
  • • 验证标志: v = FCR(â) ∈ {0, 1}
  • • 输出: QACR ∈ [0, 1]

公式推导

QACR = {

0, if v = 0 (格式无效)

(1/|valid|) Σ_{i∈valid} I(â_i = a_i), if v = 1

}

计算步骤详解

  1. 格式验证: 检查 â 是否符合 Fast+ tokenizer 的输出格式
  2. 有效位置提取: 识别动作 token 所在位置（排除特殊 token）
  3. 逐位匹配: 比较 â_i 与 a_i 在每个有效位置是否相等
  4. 准确率计算: 正确匹配数 / 有效位置总数

14.3 CTAR 公式详解

CTAR = (1/T) Σ_{t=1}^T [ω_pose · r_pose(t) + ω_gripper · r_gripper(t)]

其中 T 是动作块中的时间步数

Pose Reward (指数衰减设计)

r_pose(t) = exp(-β · ||p̂_t - p_t||)

  • • 设计原理: 小偏差应得到接近 1 的奖励，大偏差应快速衰减
  • • β=0.8: 控制衰减速率，偏差 1.0 时奖励≈0.45
  • • 平滑性: 相比阶跃函数，指数衰减提供更稠密的梯度信号

Gripper Reward (二元设计)

r_gripper(t) = I(ĝ_t = g_t)

  • • 设计原理: 夹爪开闭是离散决策，非对即错
  • • 严格性: 错误的夹爪状态会导致任务失败
  • • 与 Pose 分离: 允许独立优化，避免梯度干扰

加权组合逻辑

完整公式：

CTAR = (1/T) Σ_t [α·exp(-β·||Δp_t||) + I(Δg_t=0)]

α=5 是 pose 奖励的放大系数，β=0.8 控制衰减

设计考量：

  • • α=5 使 pose 奖励范围 [0,5]，gripper 为 {0,1}
  • • 时间平均确保不同时长动作块可比较
  • • 解码后计算，反映真实的物理执行效果

14.4 Algorithm 1: QACR 伪代码 Walkthrough

Algorithm 1: Quantized Action Consistency Reward (QACR)
Input: predicted tokens â, reference tokens a
Output: consistency reward r_qacr ∈ [0, 1]
1: function QACR(â, a):
2: // Step 1: Format validation
3: v ← ValidateFormat(â) // Check Fast+ tokenizer output format
4: if v == 0 then
5: return 0 // Invalid format gets zero reward
6: end if
7: // Step 2: Extract valid action positions
8: valid_indices ← GetActionTokenPositions(â)
9: // Step 3: Token-wise comparison
10: correct ← 0
11: for i in valid_indices do
12: if â[i] == a[i] then
13: correct ← correct + 1
14: end if
15: end for
16: // Step 4: Compute accuracy
17: r_qacr ← correct / |valid_indices|
18: return r_qacr
19: end function

关键行解读

  • Line 3-6: FCR 前置检查，确保只有格式正确的输出参与评估
  • Line 8: 动作 token 位置提取，过滤掉 prompt 中的非动作部分
  • Line 12: 严格相等匹配，离散动作空间的精确度要求
  • Line 17: 归一化确保不同长度动作块可比较

14.5 Algorithm 2: CTAR 伪代码 Walkthrough

Algorithm 2: Continuous Trajectory Alignment Reward (CTAR)
Input: predicted tokens â, reference tokens a, params α, β
Output: trajectory reward r_ctar ∈ [0, +∞)
1: function CTAR(â, a, α, β):
2: // Step 1: Decode tokens to continuous actions
3: P̂ ← FastPlusDecode(â) // Predicted trajectory: [(p̂₁,ĝ₁), ..., (p̂_T,ĝ_T)]
4: P ← FastPlusDecode(a) // Reference trajectory: [(p₁,g₁), ..., (p_T,g_T)]
5: // Step 2: Initialize trajectory reward
6: r_ctar ← 0
7: T ← Length(P)
8: // Step 3: Compute per-timestep reward
9: for t = 1 to T do
10: // Pose reward: exponential decay based on position error
11: pose_error ← ||P̂[t].position - P[t].position||₂
12: r_pose ← α · exp(-β · pose_error)
13: // Gripper reward: binary matching
14: r_gripper ← 1 if P̂[t].gripper == P[t].gripper else 0
15: // Accumulate
16: r_ctar ← r_ctar + r_pose + r_gripper
17: end for
18: // Step 4: Temporal averaging
19: r_ctar ← r_ctar / T
20: return r_ctar
21: end function

关键行解读

  • Line 3-4: Token 解码为连续动作，这是 CTAR 与 QACR 的本质区别
  • Line 11-12: Pose 使用 L2 距离，指数衰减确保小偏差得到合理奖励
  • Line 14: Gripper 严格匹配，开/关是离散决策
  • Line 19: 时间平均消除序列长度影响，使不同任务可比

15. 深度实验分析

15.1 训练动态

训练曲线分析

Step 0-100: Warmup，策略探索
Step 100-500: 快速上升期，MDPR 引导
Step 500-1000: 收敛期，性能稳定

典型收敛步数：约 800-1000 步（batch size=32）

收敛性观察

  • • 奖励方差: 早期高（探索），后期趋于稳定
  • • KL 散度: 始终保持在 0.1-0.3 之间，有效防遗忘
  • • MDPR 各分量: QACR 最先收敛，CTAR 次之，FCR 维持高位
  • • 样本效率: 平均 400-500 个 batch 达到最终 90% 性能

  • • β∈[0.5,1.0]: 性能变化 <3%，相对鲁棒

  • • λ (FCR 权重): 0.1 足够确保格式正确
  • • λ>0.2: 格式奖励过度主导

15.3 失败案例分析

类型1: 格式错误

发生频率: ~3%

症状: 输出不符合 Fast+ 格式，无法解码

原因: FCR 权重过低或训练早期探索

缓解: λ≥0.05 基本消除

类型2: 局部最优

发生频率: ~8%

症状: 策略卡在次优行为模式

原因: G 太小导致探索不足

缓解: G≥8，或增加温度参数

类型3: 灾难性遗忘

发生频率: <5% (持续学习)

症状: 学习新任务后旧任务性能下降

原因: ER 缓冲区大小不足或 KL 约束过弱

缓解: 增加 buffer size，强化 KL

rewards = torch.stack([mdpr(o, ref) for o in outputs])

11. 深度批判

深入剖析：为什么有效？

  • On-policy 的优势：使用当前策略采样，数据分布与策略一致，避免离线数据分布偏移；GRPO 的组内相对优势估计减少了方差
  • MDPR 的多维互补：QACR 解决离散空间准确性，CTAR 解决连续执行精度，FCR 解决格式稳定性；三者从语义到物理到结构全面覆盖
  • Env-free 的可扩展性：无需环境交互意味着可以大规模并行，不受 sim-to-real gap 限制
  • Plasticity-Stability 平衡：On-policy RL 天然比 SFT 更抗遗忘，结合 ER (Experience Replay) 实现持续学习

局限性质疑

  • 局限1 - 仅限于离散动作：论文明确承认只针对离散动作模型，连续动作模型性能仍有差距
  • 局限2 - 依赖参考轨迹：CTAR 需要 ground-truth 轨迹计算奖励，在真实场景中可能难以获得
  • 局限3 - 超参数敏感：MDPR 的 ω 和 λ 需要调优，不同任务可能需要不同配置
  • 局限4 - 未与其他 RL 方法充分对比：如 DPO、PPO 等在线 RL 方法的直接对比有限
