2 研究背景

📊 领域现状

动态场景重建

基于NeRF和3DGS的动态重建方法通常需要逐场景优化，通过规范空间变形或锚点变换建模运动，依赖点跟踪或光流监督，且常需3D框标注或场景图构建，计算昂贵且难以扩展。

前馈重建

DUSt3R、VGGT等方法利用大规模数据学习的先验实现快速推理，但主要关注静态场景。4D前馈重建面临运动下几何退化和运动建模两大挑战。

⚠️ 4D前馈重建的核心瓶颈

1
运动下几何退化

VGGT的交替注意力(AA)特征虽包含隐式运动线索，但主要用于静态结构理解。Page4D、VGGT4D等方法通过动态掩码放大运动线索来改善几何，但未根本解决。

2
运动建模困难

现有方法要么需要显式4D监督（数据规模有限），要么受限于AA层结构（对所有帧等权处理，无法表示定向运动）。DGGT等使用外部跟踪器，引入额外依赖。

3 核心问题定义

问题形式化

给定输入视频序列 {I₁, I₂, ..., I_T}，目标是重建动态街道场景的4D表示，包括：

  • • 每帧相机位姿 (K_t, E_t)
  • • 静态3D高斯表示 G_static
  • • 动态3D高斯表示 G_dynamic(t) 及每像素速度 v_t
传统方法
✗ 需要COLMAP/SfM预处理位姿
✗ 逐场景优化（数小时）
✗ 依赖外部跟踪器/分割模型
StreetForward目标
✓ 前馈推理（秒级）
✓ 无需位姿估计
✓ 无需外部跟踪/分割

研究假设与验证思路

  • H1: 在VGGT的AA层上添加因果掩码注意力可增强定向运动建模能力，无需显式4D监督
  • H2: 统一3DGS表示（静态+动态）结合跨帧渲染可实现时序一致的4D重建
  • H3: 局部刚性约束和前向后向一致性可稳定无监督运动学习

4 架构设计

交替注意力 (Alternating Attention)

VGGT的核心设计，通过帧间注意力和全局注意力的交替实现多帧信息融合：

帧间注意力 (Frame Attention)

每个帧内的token相互关注，捕获局部空间信息

全局注意力 (Global Attention)

跨帧token相互关注，聚合时序信息

StreetForward使用36层AA（18层帧间 + 18层全局），从Pi3预训练权重初始化。

因果动态建模

时序位置编码

在AA输出token上拼接正弦位置编码，注入显式时序信息：

Z̃_t = [Z_t ; PE(t)]
因果掩码注意力

使用4层因果掩码注意力模块（16头），限制query-key对只能来自指定的源-目标帧关系：

M[b,h,i,j] = 1 if frame(i)=f_s and frame(j)=f_t else 0

其中 f_t = f_s ± 1（前后向预测），支持±1到±4帧的因果范围。

统一3DGS表示

静态和动态内容统一用3D Gaussian Splatting表示，关键设计：

  • • 静态高斯：在整个序列中持续存在，使用不透明度稳定化正则防止塌陷
  • • 动态高斯：根据预测速度跨帧传播聚合，实现时序融合
  • • 动态掩码：运动头预测每像素动态概率M_t，用于分离静态/动态区域

5 实现细节

模型配置

图像编码器 DINOv2 (frozen)
AA层数 36 (18帧间+18全局)
因果注意力层 4层, 16头
预训练权重 Pi3 (progressive unfreeze)
运动头初始化 最终层权重置零

训练配置

数据集 Waymo Open Dataset
训练集 798 scenes (~190-200帧/场景)
时序下采样 1× - 4× 随机
训练策略 两阶段 (先静态后动态)
评估 202 scenes (test split)

两阶段训练策略

第一阶段：静态重建
  • • 禁用运动头
  • • 禁用动态高斯时序聚合
  • • 使用所有帧静态高斯 + 当前帧动态高斯
第二阶段：动态融合
  • • 激活运动头
  • • 启用动态高斯跨帧聚合
  • • 基于预测速度进行时序融合

6 数学机制

因果掩码注意力

掩码定义
M[b,h,i,j] = 1 if frame(i)=f_s and frame(j)=f_t else 0
注意力计算 (Eq. 1)
A^(h) = softmax(Q^(h)K^(h)ᵀ/√d_h + log M) V^(h)

log M将不允许的注意力位置设为-∞，实现硬性因果约束。

运动正则化

局部刚性约束 (Eq. 2-4)
L_rigidity^2D = Σ||v_i - v_j||² · w(M_i) for j ∈ N_2D(i)
L_rigidity^3D = Σ||v_i - v_j||² · w(M_i) for j ∈ N_3D(i)

鼓励图像空间和3D空间中邻近像素具有一致运动

前向后向一致性 (Eq. 5)
x_t^forward = x_{t-1} + v_{t-1}·Δt
L_consistency = ||v_forward + v_backward||²

常速度假设下，前向和后向速度应大小相等、方向相反

渲染与几何正则

RGB重建损失 (Eq. 6)
L_rgb = ||I_t - Render(G_static ∪ G_dynamic^(t))||²
不透明度稳定 (Eq. 7)
L_opacity = -Σ log(o_i) · w_i

鼓励高斯保持接近1的不透明度，防止几何错误时通过降低透明度来"隐藏"

深度一致性 (Eq. 8)
L_depth = ||D_pred - D_render||²

深度头预测与3DGS光栅化深度对齐

7 关键创新深入解析

创新点1: 因果掩码注意力

L1: 直观理解

就像人类理解运动时需要明确"从哪里来，到哪里去"，VGGT的原始AA对所有帧一视同仁，无法区分时间方向。因果掩码就像给模型戴上了"时间方向眼镜"，让它能明确建模从源帧到目标帧的定向运动。

L2: 机制流程
VGGT原始AA
Frame 1 ←→ Frame 2 ←→ Frame 3
双向、无序、无方向
StreetForward因果
Frame 1 → Frame 2 (前向)
Frame 3 → Frame 2 (后向)
单向、有方向、显式建模
L3: 技术细节
  • • 掩码位置： 在softmax之前添加log M，将禁止位置设为-∞
  • • 可变相位： 随机时序下采样(1×-4×)使因果范围±1到±4帧，增强鲁棒性
  • • 双向建模： 同时预测前向和后向速度，用于完整性验证和遮挡恢复
  • • 零初始化： 运动头最终层权重置零，确保训练初期稳定

创新点2: 跨帧动态高斯聚合

核心洞察：当前帧的动态物体应该能从相邻帧"解释"。通过将相邻帧的动态高斯按预测速度传播到当前帧，实现：

🎯
遮挡恢复

当前被遮挡的部分可从其他帧补全

🔗
时序一致性

强制同一物体在不同帧的表示一致

🚫
防止退化

排除当前帧自身动态高斯，防止错误累积

⚠️ 常见误区澄清

误区 澄清
"无需位姿但推理时需要相机参数" 相机头预测位姿是模型输出的一部分，推理时无需外部提供
"因果注意力只在训练时使用" 因果注意力是模型结构的一部分，训练和推理都使用
"动态掩码需要预训练分割模型" 动态掩码由运动头预测，无需外部分割模型
